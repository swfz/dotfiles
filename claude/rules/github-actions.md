# GitHub Actions Rules

## バージョン固定

- サードパーティアクションは+SHAで固定する（例: `actions/checkout@de0fac2e4500dabe0009e67214ff5f5447ce83dd # v6.0.2`）
- `@main` や `@master` など、ブランチ参照での指定は禁止
- メジャーバージョンのみの指定（例: `@v4`）は避け、パッチバージョンまで指定する
- 既存ワークフローで使われているアクションのバージョンを変更する場合は、必ず確認を取る

## セキュリティ

- ワークフローのトップレベルに `permissions` を明示的に指定し、最小権限の原則に従う
- `${{ }}` 式でユーザー入力を直接シェルコマンドに展開しない（コマンドインジェクション防止）
- シークレットは `secrets.*` 経由で参照し、ログに出力されないようにする
- `pull_request_target` トリガーを使う場合はセキュリティリスクを確認してから使用する

## secrets と vars の使い分け

- **`secrets.*`**: トークン、パスワード、APIキー、Webhook URLなど、ログに露出してはいけない機密値に使う
- **`vars.*`**: 環境名、フラグ、チャンク数（例: `vars.TOOLS_CHUNKS`）、リトライ回数、Feature Flag、GCPプロジェクトID、workload_identity_provider、service_accountなど、ログに出ても問題ない設定値に使う
- 迷ったら `secrets.*` を使う（varsに機密値を入れるとログに平文で出る）
- 新しい変数を追加する場合は `secrets` と `vars` どちらに格納すべきか確認を取る

## タイムアウト

- すべてのジョブに `timeout-minutes` を設定する（未設定時のデフォルトは360分＝6時間で、無駄なリソース消費につながる）
- 処理の想定所要時間に応じた適切な値を設定する
